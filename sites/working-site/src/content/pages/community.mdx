The TreeSeed fixture is for contributors who want a small but realistic tenant to read, test, and improve. It is especially useful for:

- maintainers refining shared package defaults
- writers improving contributor orientation
- developers checking whether a runtime change still feels coherent in a tenant
- operators reviewing how verification, forms, and docs fit together

The best collaboration style here is explicit and source-guided. The fixture should help people understand what TreeSeed does today, not force them to reverse-engineer it from package code alone.
