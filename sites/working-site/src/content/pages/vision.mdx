The TreeSeed fixture is not a marketing site. It is a working surface that shows how TreeSeed can organize documentation, inquiry, contributors, books, and operational guidance inside one small tenant.

Its purpose is practical:

- give contributors a generic site they can read before touching package internals
- keep the public model of questions, objectives, people, agents, and books visible
- provide a realistic test ground for `check`, `build`, and smoke verification
- make it easier to spot when package changes improve or degrade the tenant experience

TreeSeed is most useful when development and management stay close together. The fixture demonstrates that idea by keeping setup notes, workflow guidance, content examples, and verification surfaces in the same place.

If you want the current implementation posture, read the [status](/status/) page next. If you want the deeper content model, continue to [architecture](/architecture/).
