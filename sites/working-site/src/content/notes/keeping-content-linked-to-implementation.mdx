summary: The fixture should stay close enough to the package that page, content, and workflow regressions become visible quickly.
---

Every major content type in the fixture is there for a reason. If a page, book, question, objective, or form disappears from the example site, contributors lose a concrete way to observe whether the package still supports that surface cleanly.
